## BUG-001 · 冲刺中起跳被键盘鬼影吞掉

- **状态**:⏸ 暂缓(硬件限制,非代码缺陷)
- **优先级**:低
- **发现日期**:2026-06-11

### 现象
在已经按住 `W` + `Shift` 的同时再按 `空格`,跳跃不触发;松开 `Shift` 后即可正常跳。
若起跳后**不松开空格**再按 `Shift`,则不受影响。

### 复现步骤
1. 按住 `W` 前进。
2. 按住 `左Shift` 冲刺。
3. 此时按 `空格` → 跳跃无效。
4. 松开 `Shift` → 跳跃恢复。

### 根因
键盘采用矩阵扫描,`W` + `Shift` + `空格` 三键在矩阵中冲突(anti-ghosting blocking),
导致**新的**空格 keydown 事件被键盘硬件屏蔽,浏览器/JS 根本收不到该事件。
属于键盘硬件限制,纯软件无法恢复丢失的按键。

### 影响范围
仅影响"边冲刺边连续起跳"(bunny-hop 类)操作。
常见的"落地加速逃离"(空中提前按住 Shift,落地冲刺)**不受影响**,因为落地瞬间只按了
`W` + `Shift` 两键、未碰空格。

### 可选解法(择一,待定)
1. 把冲刺键从 `Shift` 改到不冲突的键(如鼠标侧键 `Mouse4`、`Q`、`F`)。
2. 移植到 Unity/原生客户端后,此类底层输入限制可用 RawInput / 手柄规避。
3. 维持现状,在说明里告知玩家用手柄或换冲刺键。

### 备注
当前冲刺已采用"按住 Shift 实时推导奔跑状态"的解耦实现,跳跃由 keydown 事件驱动,
软件层已无相关缺陷。本条仅为硬件层面遗留。
